## CRITICAL CONSTRAINTS

**Deploy workflow (never skip):**

1. Place optimized WebP files in `site/public/images/`
2. Update `The Manual/recipes.json` stage flags (heroImage, heroImageOptimized, deployed)
3. Commit and push to GitHub
4. Run `bash site/scripts/deploy.sh` from repo root

**heroImage format:** filename only — `"07-10.webp"`. Never `/images/07-10.webp`. The template prepends `/images/` automatically.

**Built recipe files:** `site/src/content/recipes/` is ephemeral — fully regenerated by `prepare-content.py`. Never hand-edit those files.

**Agent changes:** Every modification requires version bump in file header + `Agents/AGENT_CHANGELOG.md` entry + propagation to secondary surfaces.

**Build command:** Always `npm run build` from `site/` (runs astro build + pagefind together). Never `astro build` alone.

**Git:** Commit directly to main. One commit, everything bundled.
